# Getting Started

Get PAIONE running in under 5 minutes.

## Prerequisites

| | Minimum | Recommended |
|---|---------|-------------|
| CPU | 2 cores | 4 cores |
| RAM | 4 GB | 8 GB |
| Disk | 20 GB | 50 GB |
| Docker | 24.0+ | Latest |
| OS | Ubuntu 22.04, macOS 13+, Win 10+ (WSL2) | Ubuntu 24.04 |

## Installation

```bash
git clone https://github.com/oliverhees/paione.git
cd paione
cp .env.example USER/.env
```

Edit `USER/.env` and add your Anthropic API key:

```env
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

## Start PAIONE

```bash
docker-compose up -d
```

This starts all services:

| Service | Purpose |
|---------|---------|
| **paione-web** | Next.js frontend (port 3000) |
| **paione-api** | FastAPI backend (port 8000) |
| **postgres** | PostgreSQL database |
| **redis** | Cache and message broker |
| **falkordb** | Graph database for Graphiti memory |
| **celery-worker** | Background task execution |
| **celery-beat** | Scheduled task scheduler |

## Setup Wizard

Open [http://localhost:3000](http://localhost:3000). The setup wizard guides you through three steps:

1. **Welcome** -- Overview of what PAIONE is
2. **Profile** -- Create your user account (name, email, password)
3. **AI Connection** -- Enter your Anthropic API key (optional, can be added later in Settings)

After completing the wizard, you land in the chat interface.

## First Steps After Setup

1. **Chat** -- Send your first message and see PAIONE respond
2. **Settings** -- Configure your [Persona](/en/configuration#persona-files) to customize how PAIONE communicates
3. **TELOS** -- Fill in your [TELOS dimensions](/en/telos) so PAIONE knows who you are
4. **Skills** -- Explore the built-in [Skills](/en/guide/skills) or create your own
5. **Werkzeuge** -- Connect [MCP servers or REST APIs](/en/guide/werkzeuge) to give PAIONE tools
6. **Workflows** -- Set up [scheduled tasks](/en/guide/workflows) to automate recurring work

## Project Structure

```
paione/
  web/       --> Next.js frontend (dashboard, chat, settings)
  api/       --> FastAPI backend (REST API, LLM, skills, routines)
  USER/      --> Your configuration (persona, TELOS, .env)
  DATA/      --> Runtime data (storage, uploads -- gitignored)
  SYSTEM/    --> System defaults (updated with releases)
  docs/      --> This documentation portal (Nextra)
  infra/     --> Docker and Nginx configuration
```
